GPS Lycée : Navigation interne

Application web complète permettant de calculer le plus court chemin entre deux salles de mon lycée. Basée sur un algorithme BFS (Breadth-First Search) appliqué à un graphe de bâtiments et d'étages, servie par une API REST Flask.

Python Flask BFS / Graphes REST API HTML/CSS/JS JSON
  • Algorithme BFS pour trouver le chemin le plus court
  • Graphe dynamique construit depuis un fichier JSON
  • API REST avec 4 endpoints (salles, salle, chemin, bâtiments)
  • Interface avec autocomplétion en temps réel
  • Détection automatique des changements de bâtiment
WEB APP · ALGORITHMIQUE
NSI_main.py
# Algorithme BFS — plus court chemin def bfs(depart, arrivee): if depart not in GRAPHE: return None file = deque([[depart]]) vus = {depart} while file: chemin = file.popleft() noeud = chemin[-1] for voisin in GRAPHE[noeud]: if voisin not in vus: if voisin == arrivee: return chemin + [voisin] vus.add(voisin) file.append(chemin + [voisin]) return None # Construction du graphe depuis JSON def construire_graphe(data): graphe = {} for bat in data["batiments"].values(): for etage in bat["etages"].values(): ids = [s["id"] for s in etage["salles"]] # Toutes les salles d'un étage → reliées for sid in ids: graphe[sid] = set(ids) - {sid} return graphe # Route API — calcul de chemin @app.route("/api/chemin") def chemin(): dep = request.args.get("de") arr = request.args.get("a") resultat = bfs(dep, arr) return jsonify(resultat)

03 — APPROCHE

Comment je travaille.

01

Comprendre le besoin

Avant d'écrire une ligne de code, je définis clairement le problème à résoudre. Pour le GPS lycée : comment naviguer efficacement entre des salles ? Pour le site vitrine : comment convaincre un client de faire confiance à la réparation ?

02

Concevoir l'architecture

Modélisation des données (JSON pour le graphe, classes Python pour la POO), choix des technologies adaptées (Flask pour l'API, PHP pour l'authentification).

03

Implémenter proprement

Code commenté, fonctions avec responsabilité unique, gestion des erreurs (404, paramètres manquants), et validation des données. Je ne "bricole" pas.

04

Soigner l'expérience

L'interface utilisateur compte autant que le backend. Autocomplétion, messages d'erreur clairs, design responsive. Chaque détail est pensé pour que l'utilisateur final ait une expérience fluide et agréable.
